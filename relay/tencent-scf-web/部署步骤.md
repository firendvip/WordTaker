# 腾讯云 SCF「Web 函数」流式中转 部署步骤

> 这一版支持**流式**(边生成边回传),客户端即可"边说边出字"。
> 它取代旧的事件函数中转;同一个 URL 同时支持流式与非流式。

## 一、打包

本目录已含两个文件:`server.js`、`scf_bootstrap`(需可执行)。打成 zip:

```bash
cd relay/tencent-scf-web
chmod +x scf_bootstrap
zip -r wordtaker-relay-web.zip server.js scf_bootstrap package.json
```

> 一定要让 `scf_bootstrap` 在 zip 里保持可执行权限(上面的 chmod 已处理)。

## 二、新建 Web 函数

1. 腾讯云云函数控制台 https://console.cloud.tencent.com/scf → 新建
2. 选择:
   - 创建方式:**从头开始**
   - 函数类型:**Web 函数**
   - 名称:`wordtaker-relay-web`
   - 运行环境:**Nodejs 18.15**
   - 地域:国内(广州/上海)
3. 提交方式:**本地上传 zip**,选上一步的 `wordtaker-relay-web.zip`
4. 启动命令 / 监听端口:保持默认(`scf_bootstrap`,端口 **9000**)

## 三、关键:开启流式响应 + 环境变量

- 在函数配置里把「**响应模式**」设为「**流式响应 / 响应流式返回**」(没开就退化成一次性返回)。
- 「环境变量」新增(值与之前一致):
  - `DEEPSEEK_API_KEY` = 你的真实 DeepSeek key
  - `APP_TOKEN` = 之前那串访问令牌
  - 可选 `MAX_INPUT_CHARS=1000`

## 四、拿到访问地址

函数详情 →「访问路径 / 函数URL」,得到形如
`https://<id>.<region>.tencentscf.com/` 的地址,**完整发给助手**。

## 五、切到流式(助手来做)

助手会把客户端:
- `llm_relay_url` 改成这个新地址;
- `llm_streaming_enabled` 设为 `true`;
然后本地 curl 验证流式(应看到逐段 `{"d":"…"}` + 末尾 `{"done":true}`)。

## 六、保活(可选)

Web 函数同样有冷启动。可在该函数上加一个**定时触发器(每 5 分钟)**或用外部 uptime 定时 ping URL,保持容器常驻。

## 回退

如发现流式有问题,把客户端 `llm_streaming_enabled` 设回 `false` 即可——立即回到稳定的非流式整体粘贴(URL 不用改,Web 函数同时支持两种)。
