# 腾讯云云函数 SCF 部署步骤（文案优化中转）

> 目标：把中转函数挂到你的腾讯云上。真实 DeepSeek key 只作为「环境变量」存在云上，
> 不写进任何代码/文件，发出去的客户端里也没有 key —— 用户无法提取。
>
> 真实的 `DEEPSEEK_API_KEY` 和 `APP_TOKEN` 的值，由助手在对话里单独给你，**不写进本文件**。

## 一、新建函数

1. 打开腾讯云云函数控制台：https://console.cloud.tencent.com/scf
2. 左侧「函数服务」→ 顶部地域选**离你/用户近的国内地域**（如「广州」「上海」）→ 点 **新建**。
3. 选择：
   - 创建方式：**从头开始**
   - 函数类型：**事件函数**
   - 函数名称：`ququ-relay`
   - 运行环境：**Nodejs 18.15**（或 16.13，需带全局 fetch）
   - 地域：保持国内地域
4. 展开「函数代码」：把 [`index.js`](index.js) 的**全部内容**粘贴进在线编辑器，覆盖默认代码。
   - 执行入口（Handler）保持：`index.main_handler`

## 二、配置环境变量（关键：key 放这里）

在新建页往下找「**高级配置**」→「**环境变量**」，新增两条（值由助手在对话里给你）：

| Key | Value |
|-----|-------|
| `DEEPSEEK_API_KEY` | （助手给你的真实 DeepSeek key） |
| `APP_TOKEN` | （助手给你的访问令牌） |

可选（不填用默认）：`DEEPSEEK_MODEL=deepseek-v4-flash`、`MAX_INPUT_CHARS=4000`。

## 三、加 API 网关触发器（拿公网地址）

1. 同页「触发器配置」（或创建后进函数详情→「触发管理」）→ **创建触发器**。
2. 触发方式：**API 网关触发器**。
3. 请求方法：`ANY` 或 `POST`；鉴权：**免鉴权**（我们用 APP_TOKEN 自己鉴权）。
4. 创建后会得到一个**访问地址**，形如：
   ```
   https://service-xxxxxxxx-1300000000.gz.apigw.tencentcs.com/release/
   ```
   **把这个地址完整复制给助手。**

## 四、剩下的交给助手

把上面的访问地址发给助手，助手会：
- 用命令测试中转是否通；
- 填进客户端 `src/helpers/relayConfig.js` 并开启中转；
- 让客户端走中转、不再使用本地 key。

## 五、防盗刷（部署完成后建议）

- 在 DeepSeek 后台设**月度消费上限**（最硬的兜底）。
- 发现异常：改 SCF 环境变量里的 `APP_TOKEN` 并重新发版客户端即可吊销旧令牌。
